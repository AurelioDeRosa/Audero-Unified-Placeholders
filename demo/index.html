Audero Unified Placeholders Demo

Form 1 - Search form

This form shows you how, by default, Audero Unified Placeholders respects the native support of the placeholder attribute while it emulates and styles it perfectly for those that don't support it.

Form 2 - Registration form

This form shows you how you can overwrite the attribute's support in modern browsers and also apply a defined style to the placeholder. In this case, I applied the black color and the bold.

This example teach you also an important thing. Since the plugin uses the elements' value property to emulate the placeholder attribute. This means that using the usual <input type="reset" /> element to reset the fields, will delete all the values as well as the emulated placeholders. To avoid this problem, you can use the reset method of Audero Unified Placeholders. To see it in action, you can click the button called "Reset Registration form".

You can read more on this in the documentation.

Form 3 - Search number form

As said in the previous demo, this polyfill uses the value property of the elements. Chances there are that in your form you are using a plugin to validate the data inserted by the user. This means that the validator plugin will validate the values written as well as the placeholders. Another problematic case occurs when the user don't fill a field. In this circumstance, you probably want that the server will receive the field's value as empty instead of valorized with your placeholder.

This example shows how to solve these cases. The solution constists in simply disable the plugin before validate the data or sending them to the server. To achieve this goal, you can use the disable method of Audero Unified Placeholders. Once the data are verified, if some of them are wrong, you can enable again the plugin using the enable method.

Form 4 - Contact form

This example shows you how to destroy the style and the changes applied by this polyfill. The button "Destroy Contact form" uses the destroy method of the plugin to achieve this goal.
